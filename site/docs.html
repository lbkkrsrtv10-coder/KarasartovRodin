Создание и развертывание собственной криптовалюты

Полное пошаговое руководство для начинающих — от написания смарт-контракта до отправки токенов в кошельке.

01Что мы создаем

Мы создадим собственную криптовалюту (ERC-20 токен) на блокчейне Ethereum и развернем её в бесплатной тестовой сети Hoodi.

Что вы получите

  • Собственный токен с уникальным названием и символом
  • Возможность выпускать любое количество токенов
  • Возможность отправлять токены другим людям
  • Полный контроль над эмиссией

Что такое Hoodi Testnet

  • Бесплатная тестовая сеть Ethereum для разработчиков
  • Токены не имеют реальной ценности
  • Идеально подходит для обучения и тестирования

05Написание смарт-контракта

Шаг 1: Создайте файл контракта

Создайте файл contracts/Token.sol и откройте его в любом текстовом редакторе.

Шаг 2: Напишите код токена

solidity// SPDX-License-Identifier: MIT
// Версия Solidity (используем 0.8.20 для совместимости)
pragma solidity ^0.8.20;

// Импортируем стандарт ERC20 из библиотеки OpenZeppelin
import "@openzeppelin/contracts/token/ERC20/ERC20.sol";

// Объявляем контракт MyToken, который наследует все функции ERC20
contract MyToken is ERC20 {

    // INITIAL_SUPPLY = общее количество токенов
    // 1000000000 = 1 миллиард токенов
    // (10 ** 18) = 18 знаков после запятой (стандарт Ethereum)
    uint256 public constant INITIAL_SUPPLY = 1000000000 * (10 ** 18);

    // "MY_TOKEN" - название токена, "MYT" - символ (тикер)
    constructor() ERC20("MY_TOKEN", "MYT") {
        // Все токены отправляются создателю контракта
        _mint(msg.sender, INITIAL_SUPPLY);
    }
}

Шаг 3: Настройте количество токенов

Измените строку INITIAL_SUPPLY на нужное вам значение:

Желаемое количествоКод
1 000 токенов1000 * (10 ** 18)
1 000 000 токенов1000000 * (10 ** 18)
1 000 000 000 токенов1000000000 * (10 ** 18)
10 000 000 000 токенов10000000000 * (10 ** 18)
Пояснение (10 ** 18) означает 18 нулей. Это нужно, чтобы токен можно было делить на мелкие части — как копейки у рубля.

Откройте truffle-config.js и замените содержимое на:

            host: "127.0.0.1",
            port: 7545,
            network_id: "*"

            network_id: 560048,

            confirmations: 2,
            timeoutBlocks: 200,
            skipDryRun: true
        }
    },

    }

Шаг 3: Создайте файл с сид-фразой (опционально)

Важно Никогда не публикуйте файл .secret в интернете или GitHub! Добавьте его в .gitignore.

08Подготовка сети Hoodi

Hoodi — тестовая сеть Ethereum, запущенная в 2025 году на замену Ropsten и Goerli. Полностью совместима с основной сетью, бесплатна, тестовый ETH можно получить из кранов.

Добавить Hoodi в MetaMask вручную

  1. Откройте MetaMask → выпадающий список сетей → Добавить сеть
  2. Нажмите Добавить сеть вручную и заполните поля:
ПолеЗначение
Имя сетиHoodi Testnet
Новый RPC URLhttps://rpc.hoodi.ethpandaops.io
Chain ID560048
Символ валютыETH
URL обозревателяhttps://hoodi.beaconcha.in/

Или добавьте автоматически через Chainlist — найдите Hoodi и нажмите Add to MetaMask.

Получить тестовый ETH

  • Hoodi PoW Faucet — holesky-faucet.pk910.de (~0.5 ETH, 1–5 мин)
  • Google Cloud Web3 Faucet — cloud.google.com (0.1 ETH, нужен Google-аккаунт)

10Импорт токена в кошелек

  1. Переключитесь на сеть Hoodi в MetaMask
  2. Нажмите Импорт токенов (синяя кнопка внизу)
  3. В поле «Адрес контракта токена» вставьте адрес из вывода выше
  4. Символ токена и десятичные знаки (18) заполнятся автоматически
  5. Нажмите Добавить пользовательский токен → Импортировать токены
Баланс показывает 0? При использовании стандартной тестовой фразы токены принадлежат другому кошельку. Импортируйте кошелек разработчика: MetaMask → «Импортировать аккаунт» → «Приватный ключ» → вставьте ключ: 0x5de4111afa1a4b94908f83103eb1f1706367c2e68ca870fc3fb9a804cdab365a

Проверка баланса через консоль Truffle

const balance = await token.balanceOf('0x73AC80f41944e0Ebb9e3cCbf35F8de5114053E63');

.exit

Отправка через MetaMask

  1. Убедитесь, что выбрана сеть Hoodi
  2. Нажмите на токен MY_TOKEN → Отправить
  3. Введите адрес получателя и количество токенов
  4. Нажмите «Далее» → «Подтвердить»
Важно У отправителя должен быть тестовый ETH для оплаты газа.

Отправка через Hoodi Etherscan

  1. Откройте hoodi.etherscan.io и найдите адрес вашего контракта
  2. Вкладка Write Contract → Connect to Web3
  3. Функция transfer: укажите адрес получателя и сумму в минимальных единицах
  4. Пример: 100 токенов = 100000000000000000000
  5. Нажмите Write → подтвердите в MetaMask

12Устранение неполадок

ОШИБКА Mnemonic invalid or undefined
Причина

Файл .secret пустой или содержит неправильный формат.

Решение
echo "test test test test test test test test test test test junk" > .secret
ОШИБКА spawn git ENOENT
Причина

Git не установлен или не добавлен в PATH.

Решение
  1. Установите Git с git-scm.com
  2. При установке выберите «Git from the command line»
  3. Перезагрузите компьютер
ОШИБКА This version of µWS is not compatible
Причина

Проблемы с библиотекой µWS на Windows.

Решение

Это предупреждение, не влияющее на работу. Можно игнорировать.

ОШИБКА ENOENT: no such file or directory, open '.secret'
Причина

Отсутствует файл .secret.

Решение

Не критично — Truffle использует тестовую фразу. Для устранения предупреждения:

echo "" > .secret
ПРОБЛЕМА Баланс токена показывает 0 или прочерк
Причина

Токены принадлежат другому кошельку (тестовому).

Решение

Импортируйте кошелек разработчика — см. раздел Импорт в кошелек.

ПРОБЛЕМА truffle migrate зависает или очень медленный
Причина

Проблемы с RPC соединением.

Решение

Попробуйте альтернативный RPC в truffle-config.js:

"https://ethereum-hoodi-rpc.publicnode.com"

13Дальнейшие шаги

1. Добавление функции Mint (создание токенов)

solidityimport "@openzeppelin/contracts/access/Ownable.sol";

contract MyToken is ERC20, Ownable {

    uint256 public constant INITIAL_SUPPLY = 1000000000 * (10 ** 18);

    constructor() ERC20("MY_TOKEN", "MYT") Ownable(msg.sender) {
        _mint(msg.sender, INITIAL_SUPPLY);
    }

    // Только владелец может создавать новые токены
    function mint(address to, uint256 amount) external onlyOwner {
        _mint(to, amount);
    }
}

2. Добавление функции Burn (сжигание токенов)

solidity// Любой может сжечь свои токены
function burn(uint256 amount) external {
    _burn(msg.sender, amount);
}

3. Верификация контракта на Etherscan

bashnpm install @truffle/plugin-verify
truffle run verify MyToken --network hoodi --license MIT

Или вручную на hoodi.etherscan.io: найдите контракт → Verify and Publish → Solidity (Single file) → вставьте код → выберите версию 0.8.20.

4. Запуск на основной сети (Mainnet)

Осторожно Mainnet требует реальных ETH и несет финансовые риски. Развертывайте только после полного тестирования и аудита. 
javascriptmainnet: {
